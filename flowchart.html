⬡ AI News RAG

system architecture flowchart
🌐
Step 1 — Data Ingestion
TechCrunch Scraper
BeautifulSoup fetches /latest/ feed. Extracts title, date, full content, and summary from each article. Results cached to data/articles.json.
BeautifulSoup4 + Requests
✂️
Step 2 — Text Processing
Chunking
Each article split into overlapping chunks. Smaller pieces allow precise similarity matching during retrieval.
chunk_size=800 · overlap=100 · LangChain TextSplitter
🤗
Step 3 — Vectorisation
HuggingFace Embeddings API
Each chunk converted to a 384-dim vector via the HuggingFace Serverless Inference API. No local model download required.
sentence-transformers/all-MiniLM-L6-v2 · FREE API
🗄️
Step 4 — Storage
FAISS Vector Store
All chunk embeddings stored in an in-memory FAISS index. Enables millisecond-speed cosine similarity search across all articles.
FAISS (in-memory) · LangChain wrapper
▶ USER ASKS A QUESTION
🔍
Step 5 — Query Processing
Similarity Search
User question embedded via the same HuggingFace API. FAISS returns the top-k most semantically similar article chunks as context.
top-k chunks · cosine similarity
⚡
Step 6 — Generation
Groq LLM API
Retrieved chunks + user question sent to Groq. The LLM generates a grounded answer citing specific articles. Ultra-fast inference via Groq LPU hardware.
llama-3.1-8b-instant · llama-3.3-70b-versatile · FREE API
💬
Step 7 — Output
Grounded Answer + Sources
Streamlit displays the answer in the chat UI alongside source article titles. The answer is grounded in real scraped news — no hallucination.
Streamlit Chat UI · source chips
